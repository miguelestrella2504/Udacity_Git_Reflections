When would you want to create a remote repository rather than keeping all your work local?

Cuando se trabaja en equipo y se requiere que varias personas aporten al mismo repositorio, asi mismo cuando se desea 
tener un respaldo seguro en línea de la información del proyecto.

Why might you want to always pull changes manually rather than having Git automatically 
stay up-to-date with your remote repository?

Porque se podría realizar cambios que no se esperaba realizar en el proyecto al publicar o descargar
branches con cambios diferentes.

Describe the differences between forks, clones, and branches. When would you use one instead of another?

Fork es equivalente a clone, mas esto se realiza en linea en los servidores de GIT-HUB, mientras que clone se 
realiza localmente. Los branches son versiones diferentes del proyecto que pueden estar presentes en línea
o localmente, mas no son versiones del proyecto completo, sino de los cambios realizados en los archivos
especificos de este.

What is the benefit of having a copy of the last known state of the remote stored locally?

Que la puedes ver y trabajar con ella offline.

How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?

Sin git o git-hub se puede colaborar mediante respaldos en servidores en línea, mas es mas complicado,
la opción mas práctica es usar git.

When would you want to make changes in a separate branch rather than directly in master? What benefits 
does each approach have?

Es cuando se esta creando una nueva version de la aplicacion/proyecto y no se desea modificar la version 
principal hasta haber terminado y depurado la version nueva.
